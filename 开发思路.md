# 静态终端风个人主页（Version 2）开发思路

> 面向当前仓库的实现，梳理整体目标、模块划分、关键交互与可扩展方式，便于后续迭代与协作开发。

## 1. 项目定位与目标

### 1.1 定位
- 一个纯前端、纯静态的个人主页原型：左侧内容区、右侧评论区、底部“仿终端”交互区。
- 通过“终端命令”来浏览站内内容（类似在文件系统里 `ls/cd/cat`）。
- 不依赖构建工具与后端：直接用 HTML/CSS/原生 JavaScript 即可运行与部署。

### 1.2 目标
- 以极低的部署成本（静态托管即可）实现“可玩”的导航与阅读体验。
- 内容可扩展：新增页面只需要新增静态 HTML，并在“虚拟文件系统”里挂载。
- 交互稳定：命令可提示、可回放、可点击跳转，且具备打字机输出效果。
- 主题可切换：暗色/亮色两套样式，用户偏好可持久化。

### 1.3 非目标
- 不做真正的后端评论系统（当前评论为前端内存数据，刷新即失）。
- 不做复杂路由/状态管理框架（保持无依赖、可读性优先）。

## 3. 核心页面布局（壳页面）

壳页面由三块组成（见 [index.html](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/index.html#L10-L46)）：

- **内容区（Content Panel）**：展示加载后的文章/页面 HTML。
  - 标题节点：`#contentTitle`
  - 内容容器：`#contentViewport`
- **评论区（Comments Panel）**：展示评论列表。
  - 列表容器：`#commentsList`
- **终端区（Interactive Terminal）**：输入命令、显示输出。
  - 输出容器：`#terminalOutput`
  - 输入框：`#terminalInput`
  - 提示符：`#promptText`

设计要点：
- 内容区和评论区放在同一行（大屏）/同一列（小屏），通过媒体查询切换（见样式文件的 `@media (max-width: 980px)`）。
- 终端区固定在底部，占一段高度，用于“命令驱动”导航与提示。

## 4. 模块拆分与数据流

虽然代码集中在一个文件，但逻辑上拆为四个模块（见 [app.js](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js)）：

1. **内容渲染器**：负责从 URL 拉取 HTML，解析并注入到 `#contentViewport`。
2. **主题管理**：负责暗/亮切换，切换 CSS 文件，并用 `localStorage` 记住偏好。
3. **评论模块**：负责构造评论 DOM、初始化示例评论、提供 `add()` 添加新评论。
4. **终端模块**：负责命令解析、命令执行、输出渲染、历史记录、点击跳转等。

整体数据流（简化）：

```text
用户输入命令 -> tokenize() 解析 -> handleCommand() 分发
  -> cd/ls/cat/say/theme 等执行
  -> 影响：cwd(当前路径)、内容区渲染、评论列表、主题样式、终端输出
```

## 5. 内容渲染机制（为什么用 fetch + DOMParser）

内容渲染核心在 [createContentRenderer](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L28-L65)：

- 通过 `fetch(path)` 拉取目标 HTML（例如 `./post/index.html`）。
- 用 `DOMParser` 将字符串解析成文档，抽取 `body` 里的内容注入主页面。
- 额外处理：如果内容页 `body` 内第一个 `h1` 存在，将其文本作为主标题，并从内容里移除该 `h1`，避免重复显示。
- 切换动画：在注入前后给 `#contentViewport` 加/去 `switching` 类，配合 CSS 做淡入淡出（见 [styles.css](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/styles.css#L98-L107)）。

这样做的收益：
- 内容页可以是非常简单的静态 HTML 片段（只写正文即可）。
- 主页面保持统一布局与交互，内容页只关注内容本身。

注意事项：
- 若用 `file://` 直接打开，`fetch` 常会被浏览器安全策略限制；因此需要本地静态服务器或部署到静态托管（错误提示也在渲染器里做了兜底提示，见 [app.js](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L54-L61)）。

## 6. “虚拟文件系统”与站内导航

为了让终端命令像在文件系统里导航，引入了一个最小“虚拟文件系统”映射（见 [FILE_SYSTEM](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L12-L19)）：

- 一级键：目录（如 `post`、`aboutme`）
- 目录下键：文件名（如 `index.html`）
- 值：可被 `fetch` 的相对 URL（如 `./post/index.html`）

同时通过：
- `cwd: string[]` 记录当前目录路径（终端提示符由 `joinPath()` 展示）。
- `resolveDir()/listDir()` 对路径解析和列表输出（见 [app.js](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L121-L143)）。

导航策略：
- `ls`：展示当前目录内容；在根目录时展示一级目录树；非根目录时展示文件/目录列表，并为每项生成可点击的“跳转命令”链接。
- `cd`：改变 `cwd`；若进入某目录且存在 `index.html`，自动渲染该目录的 index 页面（见 [maybeRenderIndex](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L468-L477)）。
- `cat`：把“文件名”解析为映射 URL，然后渲染到内容区（见 [runCat](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L457-L466)）。

## 7. 终端交互设计

终端模块核心在 [createTerminal](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L221-L641)，其设计要点：

### 7.1 输出体验
- **打字机效果**：`typewriter()` 将输出按行逐字打印，增强“终端感”。
- **即时输出**：对简单输出使用 `appendLineInstant()`，避免不必要的等待。
- **可点击跳转**：`ls` 输出里将目录/文件渲染成 `.terminalJump`，点击后自动执行对应命令（见 [app.js](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L619-L628)）。

### 7.2 输入体验
- 维护命令历史：`ArrowUp/ArrowDown` 浏览历史（见 [onKeyDown](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L593-L615)）。
- 输出进行中禁止输入：用 `isPrinting` 防止并发输出造成 UI 混乱。

### 7.3 命令体系
- 命令清单由 `CHEAT_SHEET` 定义，`help` 将其格式化输出（见 [printHelp](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L361-L364)）。
- `handleCommand()` 统一解析与分发（见 [handleCommand](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L564-L587)）。

目前支持命令：
- `help`：查看命令说明
- `clear`：清屏
- `ls`：列出目录/文件
- `cd [path]` / `..`：进入目录/返回上级
- `cat [file]`：打开并渲染页面
- `say [text]`：新增一条评论
- `theme dark|light`：切换主题
- `sudo`：彩蛋（锁定输入并输出一段文本）

## 8. 主题切换机制

主题采用“两套 CSS 文件 + data 属性”组合：

- `documentElement.dataset.theme` 记录当前主题（`dark|light`）。
- `#themeStylesheet` 的 `href` 指向 `styles.css` 或 `styles.light.css`（见 [setTheme](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L71-L80)）。
- 用户偏好保存在 `localStorage["theme"]`，初始化时读取并应用（见 [DOMContentLoaded 初始化](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L643-L648)）。

设计收益：
- 两套样式完全静态，切换代价低且容易调试。
- 不需要 CSS-in-JS 或复杂构建流程。

## 9. 评论模块设计

评论模块以“静态示例 + 运行时新增”为主（见 [setupComments](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L196-L219)）：

- 初始生成 6 条示例评论（随机昵称 + 固定文本 + 日期）。
- `say` 命令调用 `window.__comments.add(text)`，插入到列表最前面（见 [runSay](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L519-L524)）。

边界说明：
- 评论数据当前不持久化；刷新页面会重置。
- 若未来要持久化，最小改动路径：用 `localStorage` 存数组并在 `setupComments()` 初始化时读取恢复。

## 10. 本地开发与部署方式

### 10.1 本地开发
由于内容加载依赖 `fetch`，推荐用任意静态服务器启动项目目录，例如：
- VS Code Live Server 插件
- Python：在项目根目录运行 `python -m http.server 8000`
- 任意静态托管工具均可

访问：
- `http://localhost:8000/`（以实际端口为准）

### 10.2 部署
该项目为纯静态资源，可直接部署到：
- GitHub Pages
- 任意对象存储 + 静态网站托管
- Nginx/Apache 静态目录

## 11. 扩展与迭代建议（按最小改动原则）

### 11.1 新增页面/分类
目标：让终端可导航到新内容。

步骤（推荐做法）：
1. 新建目录（例如 `projects/`），新增静态内容页（例如 `projects/index.html`）。
2. 在 [FILE_SYSTEM](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L12-L19) 中增加映射：
   - 新增一级目录键 `projects`
   - 目录下新增 `"index.html": "./projects/index.html"` 或更多文件
3. 如希望默认进入某页，修改 [DEFAULT_PAGE](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/app.js#L21)。

### 11.2 新增终端命令
目标：扩展交互能力（例如 `open`, `search`, `whoami`）。

最小改动路径：
1. 在 `CHEAT_SHEET` 追加命令说明，保证 `help` 自动展示。
2. 在 `handleCommand()` 增加分支并实现对应 `runXxx()` 方法。
3. 避免命令执行中直接操作 DOM 多处：优先复用现有 `renderPath`、`typewriter`、`appendLineInstant`。

### 11.3 内容页写作规范（建议）
- 内容页以 `h1` 开头作为页面标题（渲染器会提取并同步到主标题区域）。
- 其余内容使用标准 HTML（`p`, `ul`, `div` 等）；尽量复用现有的 `.contentHint`、`.sectionTitle`、`.tableWrap`、`.cheatSheet` 等类，保持一致风格（参考 [post/index.html](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/post/index.html)）。

## 12. 常见问题排查

### 12.1 直接双击打开后内容加载失败
现象：内容区显示“Failed to load content... file:// ... run a local static server”提示。

原因：
- 浏览器对 `file://` 下的 `fetch` 有跨域/安全限制。

解决：
- 使用本地静态服务器运行（见“本地开发”）。

### 12.2 主题切换无效
检查点：
- `index.html` 是否存在 `id="themeStylesheet"` 的 `<link>`（见 [index.html](file:///c:/Users/zcb19/Desktop/TDPB/开发/Version%202/index.html#L7)）。
- 是否能写入 `localStorage`（无痕模式/隐私设置可能限制）。

## 13. 迭代路线（可选）

若后续要把“可玩”升级为“可用”，建议按优先级：
1. 评论持久化（localStorage）与输入校验（长度/敏感词等）
2. 内容索引（生成文章列表、最近更新、分类导航）
3. `search` 命令（基于内容索引做简单搜索）
4. 内容页支持 Markdown（构建期转换或运行时转换，需引入依赖与安全策略）
